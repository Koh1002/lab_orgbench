# ビジネス提案評価ルーブリック

各次元を1-5のスケールで評価してください。

## 1. 実現可能性 (feasibility)
- **5**: 具体的な実装計画、現実的なタイムライン、明確なリソース要件が全て揃っている
- **4**: 大部分が具体的だが、一部に曖昧さがある
- **3**: 基本的な計画はあるが、具体性に欠ける部分が多い
- **2**: 計画が非現実的、または重要な要素が欠落している
- **1**: 実現可能性の検討がほとんどない

## 2. 新規性 (novelty)
- **5**: 独自の視点・アプローチがあり、既存ソリューションとの明確な差別化がある
- **4**: ある程度の独自性があるが、一部は既存アプローチの改良
- **3**: 標準的なアプローチ。特に目新しい要素はないが、適切
- **2**: ほぼ既存ソリューションの焼き直し
- **1**: 独自性が全くない

## 3. 市場洞察 (market_insight)
- **5**: 定量的な市場データ、競合分析、顧客セグメント分析が充実。具体的な企業名・数値あり
- **4**: 市場分析は充実しているが、一部のデータが不足
- **3**: 基本的な市場理解はあるが、データの具体性に欠ける
- **2**: 市場分析が表面的。具体的なデータがほとんどない
- **1**: 市場分析がない、または完全に的外れ

## 4. 財務的厳密性 (financial_rigor)
- **5**: 詳細な収益モデル、コスト構造、損益分岐点、3年予測が全て整合的
- **4**: 財務分析は充実しているが、一部の仮定が不明確
- **3**: 基本的な財務計画はあるが、詳細さが不足
- **2**: 財務分析が非常に表面的、または数値に整合性がない
- **1**: 財務分析がない

## 5. 技術的深度 (technical_depth)
- **5**: 具体的な技術スタック、アーキテクチャ、実装ロードマップが明確
- **4**: 技術面の理解は深いが、一部の詳細が不足
- **3**: 基本的な技術理解はあるが、具体性に欠ける
- **2**: 技術面の理解が浅い、または不正確
- **1**: 技術面の言及がほとんどない

## 6. 総合品質 (overall_quality)
- **5**: 全セクションが高品質で一貫性があり、説得力のある提案
- **4**: 全体的に良質だが、一部に改善の余地がある
- **3**: 平均的な品質。基本要件は満たしている
- **2**: 品質にばらつきがあり、説得力に欠ける
- **1**: 全体的に品質が低い
