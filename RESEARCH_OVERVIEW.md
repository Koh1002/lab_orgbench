> **一言**: AIエージェントの「組織の作り方」を、実証・理論・適応の3段階で科学する研究
>
> 最終更新: 2026-03-15
> 著者: Koh Shinoda

## 1. 研究の全体ストーリー

### 1.1 問題意識

2025-2026年、LLMベースのマルチエージェントシステム（MetaGPT、ChatDev、CrewAI、AutoGen等）が急速に普及している。これらのシステムは「CEO」「エンジニア」「レビュアー」といった組織的メタファーを採用し、エージェント間のメッセージ経路・権限配分・レビュー構成を暗黙に設計している。

**しかし、その「組織の作り方」が出力品質にどう影響するかは、誰も体系的に調べていない。**

### 1.2 なぜLLMエージェントで組織を研究するのか

人間の組織論（Galbraith, Mintzberg, Lawrence & Lorsch等）は60年の歴史があるが、**人間組織では制御実験が不可能**。LLMエージェントにはこれらの交絡因子がない。残るのは**情報フロー設計パターン**の純粋な効果のみ。

### 1.3 3本の論文の関係

```
論文0 (Phase 0): 事実の発見      →「組織の作り方で品質は変わるか？」→ 変わる
    ↓ なぜ？
論文1 (Phase 1): メカニズムの解明 →「なぜ深い階層は悪いのか？」→ 情報の遮断
    ↓ では直せるか？
論文2 (Phase 2): 動的適応        →「組織は自分で構造を変えられるか？」
```

### 1.4 核心的発見（Phase 1 で確立）

> **LLMマルチエージェント組織は情報「保存」パイプラインではなく、情報「変換」ネットワークである。**
> **深い階層が品質を落とす真の原因は、中間層（TL）が下流エージェントの変換能力を「遮断」すること。**
> **設計目標は distortion の最小化ではなく、augmentation - distortion の最大化。**

## 3. 論文1 (Phase 1): Information Flow Analysis — メカニズムの解明 ✅

### 3.1 概要

| 項目 | 内容 |
|------|------|
| タイトル | The Information-Theoretic Cost of Hierarchy: A Channel-Theoretic Analysis of LLM Multi-Agent Organizations |
| **状態** | **論文ドラフト完成、投稿準備中** |
| ターゲット | AAMAS 2027 / NeurIPS 2027 |
| コード | `Phase1/src/orgbench/` |
| 論文 | `Phase1/paper/main.tex` |

### 3.2 核心的問い

> なぜ深い階層は品質を落とすのか？

### 3.3 8幕のストーリー

**第1幕（謎）**: OrgBenchで「浅い方が良い」と判明。だがなぜか分からない。

**第2幕（調査ツール）**: エージェントを情報チャネルとしてモデル化。タスクに10個のトレーサー（架空事実）を注入して情報の生存を追跡。

**第3幕（犯人特定）**: **TL（中間管理層）が1ステップで情報の71-97%を破壊**。4つの独立指標（トレーサー71%、数値91%、エンティティ97%、トークン65%）が同じ犯人を指す。

**第4幕（二重メカニズム）**: pass-through実験で分離:
- TLのプロンプトが過度にフィルタリング → D_filter（プロンプト修正で改善可能）
- LLMチャネル固有の歪み → D_noise（~7%/step、構造変更でのみ改善）

**第5幕（情報の回復）**: flat_mesh（TLなし）ではWriterがトレーサーを**0.30→0.70に回復**。anchor（TLあり）では0.10のまま。**TLは情報を失うだけでなく、下流の情報アクセスを遮断**している。

**第6幕（脱出口）**: 高性能モデル（Haiku TL: CR=0.87）ならdepth penaltyが消える。GPT-4o-mini TL: CR=0.35 → penalty深刻。

**第7幕（パラドックス）**: 高品質な出力はタスクプロンプトから意味的に**遠い**（r=-0.73）。良い組織は情報を保存するのではなく**変換**する。

**第8幕（設計原理）**: 5つのガイドライン:
1. 主経路のdepthを最小化 or 高容量中継エージェント
2. augmentationモードのエージェント（CEO）に最大の情報を届けよ
3. meshは主経路エージェントに容量があれば有効
4. レビューは専門次元を選択的に改善（CFO→財務, CDO→技術）
5. 低不確実性タスクほどdepth penaltyが大きい

### 3.4 主要な実証結果

| 仮説 | 結果 | キー数値 |
|------|------|---------|
| H1: Depth-Distortion | ✅ | per-run r=-0.458***, -0.40 OQ/step |
| H2: Depth vs TL filtering | ✅ | pass-through TL: 品質回復 |
| H3: モデル容量依存 | ✅ | Haiku(depth=4)=flat_hub(depth=3)=3.22 |
| H4: Mesh飽和 | ✅ | CFO CR=0.06(飽和)だが品質無影響 |
| H5: レビュー選択性 | ✅ 部分 | Tech Depth +0.37, Financial Rigor +0.57 |
| H6: CEO augmentation | ✅ | r=+0.317*** |

## 4. 論文2 (Phase 2): Dynamic Organizational Adaptation — 問題の解決

### 4.1 概要

| 項目 | 内容 |
|------|------|
| タイトル案 | Self-Reconfiguring Agent Organizations: Information-Theoretic Triggers for Dynamic Structural Adaptation in LLM Multi-Agent Systems |
| 状態 | 計画段階（Phase 1の知見を基盤に設計更新中） |
| ターゲット | ICLR 2028 / NeurIPS 2027 Workshop |
| コード | `Phase2/` (未着手) |

### 4.2 核心的問い

> LLMエージェント組織は、タスクに応じて自分で構造を変えられるか？ そして固定構造を上回れるか？

### 4.3 Phase 1 からの接続（更新済み）

Phase 1 で確立した「情報変換ネットワーク」の知見が、Phase 2 の設計を根本的に変える:

**旧計画**（Phase 1 以前）: 「information distortion を検知したら構造を変更」
**新計画**（Phase 1 の知見を反映）: 「**net transformation (A - D) を最大化**するように構造を動的に変更」

| Phase 1 の知見 | Phase 2 での活用 |
|---------------|-----------------|
| TLが情報を「遮断」する | **適応トリガー1**: TLのCRが閾値以下 → TLを排除 or pass-throughに切替 |
| 高容量モデルはdepth penaltyなし | **適応アクション**: 低容量エージェントを検出 → 高容量モデルに置換 |
| flat_meshでWriterが情報を回復 | **適応トリガー2**: 下流エージェントのTSRが低い → 直接通信チャネル追加 |
| CEO input ↑ → quality ↑ | **適応原理**: augmentationモードのハブに最大の情報を集める |
| レビューは次元選択的 | **適応アクション**: 品質次元が不足 → 該当専門レビュアーを動的追加 |
| 低不確実性タスクはdepth penaltyが大 | **適応トリガー3**: タスク不確実性を推定し、低不確実性なら浅い構造を選択 |

### 4.4 適応メカニズム（Phase 1 の知見で更新）

**適応トリガー**（情報フロー指標ベース）:

| トリガー | 検出指標 | アクション |
|---------|---------|-----------|
| **情報遮断** | critical-pathエージェントのCR < 0.3 | 該当エージェントを排除 or pass-throughに |
| **情報飢餓** | CEO input tokens < 閾値 | mesh通信に切替 or 中間層バイパス |
| **飽和** | non-criticalエージェントのCR < 0.1 | hub通信に切替（飽和回避） |
| **次元不足** | 特定品質次元が低い | 該当専門レビュアーを追加 |
| **変換不足** | final output のSSD > 0.75（タスクに近すぎ） | augmentationエージェントを追加 |

**メタコントローラー**:
- **Option 1 (主)**: Rule-Based — Phase 1 の閾値（CR < 0.3 → 遮断、CR < 0.1 → 飽和）に基づく決定木
- **Option 2 (副)**: LLM-Based — メタエージェントがステップごとのTSR/CR/SSDを監視し適応判断

### 4.5 実験計画

- **ベースライン**: flat_hub（Phase 0/1 のベスト固定構造）, flat_mesh, Oracle Adaptation
- **タスク設計**: 途中で複雑度が変化するシナリオ
  - Multi-phase: リサーチ→統合→レビューで最適構造が異なる
  - Domain shift: テーマ途中で予想外の領域が必要になる
  - Complexity escalation: 簡単な調査→技術的に困難な設計
- **評価**: 既存6次元品質 + Adaptation Frequency/Effectiveness/Net Benefit

### 4.6 なぜこれがトップ会議に通るか

- **Phase 1 の知見に基づく理論駆動の適応**: 場当たり的なヒューリスティクスではなく、「情報遮断の検知→構造変更」という原理
- **Novelty**: LLMエージェント組織の自己再構成は未開拓
- **強力なベースライン**: Phase 0/1 の固定構造ベストが比較対象
- **「情報変換ネットワーク」パラダイム**: 適応の目標が「distortion最小化」ではなく「net transformation最大化」

## 8. 「あなた何の研究をしているの？」への回答

### 30秒版

AIエージェントを複数組み合わせて仕事をさせるとき、**中間管理層（TL）が1ステップで情報の71-97%を破壊する**ことを発見しました。しかもTLは情報を失うだけでなく、**下流のエージェントが元の情報にアクセスすることを遮断**しています。TLを除いた flat 組織ではWriterがトレーサー情報を70%まで回復できるのに、TLを挟むと10%のまま。さらに意外なことに、**良い出力はタスクプロンプトから意味的に「遠い」**（r=-0.73）。これはAI組織の目的が情報の「保存」ではなく「変換」であることを意味します。

### 3分版

LLMマルチエージェントシステムで「浅い階層が深い階層より良い」ことは経験的に知られていましたが、なぜかは不明でした。

私はタスクに10個の架空事実（トレーサー）を埋め込んで、各エージェントステップで何が生き残るかを追跡しました。結果、**TL（中間管理層）が1段で情報の71%を破壊**していることが分かりました。数値は91%消失、エンティティは97%消失 — 4つの独立指標が全て同じ犯人を指しています。

さらにpass-through実験（TLに「そのまま転送せよ」と指示）で、品質低下の原因を**TLのプロンプト設計（改善可能）**と**LLMチャネル固有の歪み（~7%/step、不可避）**に分離しました。

最も意外な発見は、flat_mesh組織ではWriterがトレーサーを**0.30→0.70に回復**すること。TLが情報を「失う」だけでなく「遮断」しているため、TLを排除すると下流エージェントが元の情報に直接アクセスでき、情報を再構成できるのです。

そして高品質な出力はタスクプロンプトから意味的に**遠い**（r=-0.73）。これは**良いAI組織は情報を保存するパイプラインではなく、情報を変換するネットワーク**であることを意味します。設計目標は「情報損失の最小化」ではなく「augmentation（知識付加） - distortion（歪み）の最大化」です。

次のステップとして、この知見を使って**タスク実行中にAI組織が自動的に構造を最適化する**仕組みを構築します。TLの情報遮断を検知したら即座にTLを排除し、CEOへの入力を増やすといった、**情報フロー指標に基づく動的適応**です。
