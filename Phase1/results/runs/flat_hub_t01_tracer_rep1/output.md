# SmartKeiri Pro ビジネス提案（最終版）

## エグゼクティブサマリー

SmartKeiri Proは、日本の中小企業向けに特化した次世代会計ソフトウェアです。市場規模600億円（2022年度）から900億円（2025年度予測）への成長機会を捉え、AI駆動の自動仕訳機能と直感的なUIにより、従業員数10～100人の中小企業における会計業務の効率化を実現します。

**財務目標:**
- 初年度売上：6億円（顧客数2,000社）
- 3年目売上：15億円（顧客数6,000社）
- 初期投資：5億円
- 損益分岐点：初年度内に達成見込み

**主要な差別化要因:**
1. 中小企業特化型の機能設計
2. AI自動仕訳システム（精度96%以上を目指す）
3. 業界別テンプレート機能
4. リアルタイム財務分析ダッシュボード

## 1. 市場分析

### 1.1 市場規模と成長性

| 年度 | 市場規模 | 成長率 | 主要成長要因 |
|------|--------|--------|-----------|
| 2022年 | 600億円 | - | - |
| 2023年 | 648億円 | 8% | クラウド化推進 |
| 2024年 | 700億円 | 8% | DX支援政策 |
| 2025年 | 900億円 | 8-10% | 中小企業デジタル化加速 |

**成長ドライバー:**
- 政府のDX支援補助金（IT導入補助金など）による中小企業のデジタル化推進
- クラウドベース会計ソフトの認知度向上
- 消費税軽減税率対応やインボイス制度への対応需要

### 1.2 競合分析

| 競合企業 | 主要製品 | ターゲット | 強み | 弱み |
|---------|--------|----------|------|------|
| 弥生株式会社 | 弥生会計 | 小規模～中堅企業 | 圧倒的シェア、サポート充実 | UI/UXが旧式、AI機能不足 |
| freee株式会社 | freee会計 | スタートアップ～中堅 | モダンUI、API連携 | 小規模企業には機能過剰 |
| マネーフォワード | MF クラウド会計 | 全規模対応 | 多機能、連携機能豊富 | 価格帯が高い |

**SmartKeiri Proの差別化:**
- **中小企業特化**: 10～100人企業の実務ニーズを徹底分析した機能設計
- **AI自動仕訳**: 業界別学習データにより、高い精度を実現
- **シンプルな価格体系**: 月額2,500円からの段階的料金設定

### 1.3 ターゲット市場

**主要ターゲット:**
- 従業員数：10～100人
- 業種：製造業、卸売業、小売業、建設業、サービス業
- 特性：会計システム未導入またはオンプレミス型システム利用中
- 課題：会計業務の手作業負担、決算期の業務集中、経営判断の遅延

**市場規模推定:**
- 対象企業数：約180万社（日本の中小企業の約30%）
- 初期段階での獲得目標：2,000社（0.1%）

## 2. 技術的アプローチ

### 2.1 技術スタック（詳細版）

#### バックエンド
- **言語**: Python 3.10+
- **フレームワーク**: Django 4.2 + Django REST Framework
- **選定理由**: 
  - AI/ML統合が容易（TensorFlow、scikit-learnとの親和性）
  - 高速開発が可能（プロトタイピングから本番環境への移行がスムーズ）
  - セキュリティライブラリが充実

#### フロントエンド
- **言語**: TypeScript
- **フレームワーク**: React 18+ + Redux Toolkit
- **UI フレームワーク**: Material-UI v5
- **選定理由**:
  - 複雑な状態管理が必要な会計システムに適合
  - リアルタイム更新機能の実装が容易
  - ユーザーテスト結果から直感的なUI/UXが実現可能

#### データベース
- **主DB**: PostgreSQL 15
- **キャッシュ層**: Redis（セッション管理、キャッシング）
- **選定理由**:
  - 金融データの整合性要件を満たす（ACID特性）
  - JSON型対応により柔軟なスキーマ設計が可能
  - 監査ログ機能が充実

#### AI/ML
- **AIフレームワーク**: TensorFlow 2.13 + Keras
- **モデル**: カスタム構築のニューラルネットワーク
- **言語処理**: BERT（日本語特化モデル）の活用

### 2.2 AI自動仕訳システムの詳細

#### アーキテクチャ
```
入力（領収書/請求書）
    ↓
OCR処理（テキスト抽出）
    ↓
自然言語処理（NLP）
    ↓
AIモデル推論（仕訳分類）
    ↓
ユーザー確認・フィードバック
    ↓
モデル再トレーニング
```

#### トレーニングデータ戦略
- **初期データセット**: 
  - 公開会計データ（政府統計）：約50万件
  - パートナー企業からの匿名化データ：約30万件
  - 合計：約80万件の仕訳データ
  
- **データ収集プロセス**:
  - ベータテスト段階でユーザーから同意取得
  - 機密情報は完全に匿名化・マスキング
  - 差分学習により継続的にモデル精度を向上

#### 精度目標
- **初期リリース時**: 92%以上の自動仕訳正確率
- **6ヶ月後**: 95%以上
- **12ヶ月後**: 96%以上

#### 誤動作対策
- 信頼度スコア機能：自信度が低い仕訳は必ずユーザー確認を要求
- 業界別モデルの分離：業種ごとに特化したモデルを構築
- 継続的なQAテスト：毎月1,000件以上の新規データでテスト
