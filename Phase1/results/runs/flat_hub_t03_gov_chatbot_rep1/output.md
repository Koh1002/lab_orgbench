# 自治体向けAI住民対応チャットボット「CivicAI」 最終ビジネス提案書

---

## 1. エグゼクティブサマリー

### ビジネス機会
日本の1,741自治体は年間約2.5億件の住民問い合わせに対応しており、人口減少による職員不足が深刻化しています。政府のDX推進により自治体DX予算は2023年度1,200億円から5年で3,000億円へ拡大予定であり、AIチャットボット導入率は36.5%（2023年3月）で、未導入自治体が64%を占める大きな市場機会が存在します。

### 提案内容
当社は、自治体向けAIチャットボット「CivicAI」を開発・提供します。本サービスにより、自治体の問い合わせ対応業務を自動化し、住民サービスの24時間対応化と職員の業務効率化を実現します。

### 財務目標
- **初年度（2024年度）**: 売上2.5億円、営業利益-1.0億円（投資段階）
- **2年目（2025年度）**: 売上10億円、営業利益2.0億円
- **3年目（2026年度）**: 売上20億円、営業利益8.0億円

### 導入目標
初期ターゲット市場410自治体（大都市・中核市・一般市）において、3年目に30%（156自治体）の導入を達成し、その後5年目に60%（246自治体）の導入を目指します。

## 3. 技術的アプローチ

### コア技術スタック

#### 自然言語処理（NLP）層
- **フレームワーク**: spaCy（軽量で高速）+ NLTK（テキスト分析補助）
- **生成AI**: OpenAI API（GPT-4）の統合 + ローカルLLM（Llama 2）のハイブリッド運用
  - 公開情報の回答：OpenAI API（高精度）
  - 機密情報の回答：ローカルLLM（セキュリティ重視）
- **日本語特化**: MeCab + NEologd（固有表現抽出、自治体用語対応）

#### 機械学習・意思決定層
- **フレームワーク**: TensorFlow 2.x + PyTorch（必要に応じて）
- **モデル**: BERT系の日本語学習済みモデル（tohoku-nlp/bert-base-japanese）
- **意図分類**: マルチクラス分類モデル（自治体業務の100+ カテゴリに対応）
- **信頼度スコアリング**: 回答の信頼度を0-100で算出し、低信頼度時は人間へのエスカレーション

#### データベース・ストレージ層
- **メインDB**: PostgreSQL 14+（JSONB対応で柔軟なスキーマ対応）
- **キャッシュ**: Redis（高速応答のための会話キャッシュ）
- **ドキュメントストア**: Elasticsearch（自治体の条例・規則の全文検索）
- **ファイルストレージ**: AWS S3（ドキュメント管理、バージョン管理）

#### インフラストラクチャ層
- **クラウドプロバイダ**: AWS（国内リージョン：東京）
- **コンテナ化**: Docker + Kubernetes（EKS）
  - 自動スケーリング対応（同時接続数に応じた動的スケール）
  - 負荷分散：AWS Application Load Balancer
- **API ゲートウェイ**: AWS API Gateway + Lambda（マイクロサービス化）
- **監視・ログ**: CloudWatch + ELK Stack（Elasticsearch, Logstash, Kibana）
- **CD
