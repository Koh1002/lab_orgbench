## 0. 出発点

Modified DPI (論文 Eq.2):
```
I(X; Yₖ) = I(X; Yₖ₋₁) - Dₖ + Aₖ
```

ここで:
- X = 原タスク入力（テーマ説明 + 要件 + 制約条件）
- Yₖ = ステップk のエージェント出力
- I(X; Yₖ) = X と Yₖ の間の相互情報量（「Yₖ が X についてどれだけ知っているか」）
- Dₖ ≥ 0: ステップk での**情報歪み**（損失）
- Aₖ ≥ 0: ステップk での**情報付加**（獲得）

**注意**: I(X; Yₖ) は「Yₖ と Yₖ₋₁ の類似度」ではなく、「Yₖ が**元のタスク X**についてどれだけ情報を持っているか」である。この区別が重要。

---

## 1. 「情報」の分解: Task Information Units

### 1.1 定義

相互情報量 I(X; Yₖ) を直接計算するのは高次元テキストでは困難。そこで、X に含まれる情報を**離散的なユニットに分解**し、追跡可能にする。

**Definition (Task Information Unit)**:
タスク入力 X に含まれる、最終出力の品質に寄与しうる個別の情報単位を **Task Information Unit (TIU)** と呼ぶ。

TIU は以下の4カテゴリに分類される:

| カテゴリ | 記号 | 例 | 検出方法 |
|---------|------|-----|---------|
| **Requirements** (要件) | R | 「6セクションを含めよ」「3年間の財務予測」 | テーマYAMLのtask_promptから抽出 |
| **Facts** (事実) | F | 「ターゲットは中小企業（従業員50-300名）」 | タスク内の具体的主張 |
| **Constraints** (制約) | C | 「初期投資5,000万円以内」 | タスク内の条件文 |
| **Queries** (暗黙の問い) | Q | 「市場規模は？」（market analysisセクション要件から暗示） | 要件から推論される情報要求 |

タスク X の TIU 集合を T(X) = {t₁, t₂, ..., tₘ} とする。

### 1.2 TIU の操作的同定

**方法A: 手動アノテーション**
研究者がタスクプロンプトを読み、TIU を列挙する。5テーマ × 10-15 TIU/テーマ = 50-75 TIU。

**方法B: トレーサー注入（実験C）**
人工的に作成したTIU（架空固有名詞、架空数値等）をタスクに埋め込む。検出が確実。

**方法C: LLMベース抽出**
LLMにタスクプロンプトを与え、「この入力に含まれる個別の情報要件・事実・制約を列挙せよ」と指示。

**推奨**: 方法Aで「自然TIU」を定義 + 方法Bで「トレーサーTIU」を追加。二つの測定系を持つことで頑健性を確保。

## 2. D_k の定義

### 2.1 形式的定義

**Definition (Information Distortion)**:

ステップk でのDₖ は、**入力 Yₖ₋₁ に含まれていたが出力 Yₖ に含まれなくなった TIU の数**（正規化）:

```
Dₖ = |Preserved(k-1) \ Preserved(k)| / |T(X)|
```

ここで:
```
Preserved(k) = { t ∈ T(X) : t は Yₖ に含まれている }
```

「含まれている」の判定には3つのレベルがある:

| レベル | 名称 | 判定方法 | 例 |
|--------|------|---------|-----|
| L1 | **Literal preservation** | 完全一致 or 数値±10% | 「3,847億円」→「3,847億円」 ✓ |
| L2 | **Semantic preservation** | 意味的に同等（言い換え可） | 「3,847億円」→「約3,800億円規模の市場」 ✓ |
| L3 | **Functional preservation** | 要件を満たす内容が存在 | 「市場規模を示せ」→ 何らかの市場規模データあり ✓ |

**操作的には**:
- M7 (Tracer Survival Rate) は **L1レベルの Dₖ** を測定
- M5 (Entity Preservation Rate) は **L1-L2レベルの Dₖ** を測定
- M3 (Semantic Similarity Decay) は **L2-L3レベルの Dₖ** の代理指標

### 2.2 Dₖ の分解

Dₖ はさらに2つの成分に分解できる:

```
Dₖ = Dₖ_filter + Dₖ_noise
```

- **Dₖ_filter**: エージェントが**意図的に**除外した情報（フィルタリング）
  - 例: TLがCEOの指示から関連部分だけを選びResearcherに渡す
  - これは「有用な圧縮」であり、必ずしも品質低下につながらない

- **Dₖ_noise**: エージェントが**意図せず**失った情報（ノイズ）
  - 例: 長い入力の末尾に重要情報があったがattention不足で無視された
  - これは純粋な情報損失であり、品質を低下させる

**C1との接続**: TLのCR=0.35は65%のDₖを示すが、そのうちDₖ_filterとDₖ_noiseの比率は不明。もしTLが「不要な情報を賢くカット」しているなら、Dₖ=0.65でも品質への影響は小さいはず。しかしC1で TL CR vs OQ に正の相関（r=+0.169）があることは、TLのフィルタリングが**過度**（Dₖ_noiseが無視できない）であることを示唆。

### 2.3 Dₖ の理論的上界

エージェントがpure relay（完全中継）の場合、Dₖ = 0。
エージェントが完全に入力を無視する場合、Dₖ = |Preserved(k-1)| / |T(X)|。

**LLMチャネルの場合の推定**:
C1の結果から、TLでの定率フィルタリングモデルを仮定すると:

```
Preserved(k) ≈ α · Preserved(k-1)    where α ≈ CR ≈ 0.35 (for TL)
```

これは Dₖ = (1-α) · |Preserved(k-1)| / |T(X)| を意味し、**各ステップで入力の(1-α)の割合が失われる**という指数減衰:

```
|Preserved(n)| / |T(X)| ≈ α^n
```

C1でdepthが最強の予測変数だった事実と整合する。

## 3. A_k の定義

### 3.1 形式的定義

**Definition (Information Augmentation)**:

ステップk でのAₖ は、**入力 Yₖ₋₁ に含まれていなかったが出力 Yₖ に新たに出現した、X に関連する情報**（正規化）:

```
Aₖ = |Augmented(k)| / |T(X)|
```

ここで:
```
Augmented(k) = { 情報単位 u : u ∈ Content(Yₖ) ∧ u ∉ Content(Yₖ₋₁) ∧ u は X に関連 }
```

### 3.2 Aₖ の源泉

Aₖ の情報は3つの源泉から来る:

| 源泉 | 記号 | エージェント例 | 測定方法 |
|------|------|--------------|---------|
| **Prior knowledge** | Aₖ_prior | CEO（学習済み知識から市場動向を補完） | 出力の新規entity - 入力のentity - tool結果のentity |
| **External tools** | Aₖ_tool | Researcher（Web検索結果） | tool呼び出し結果に含まれるentityの最終出力への反映率 |
| **Reasoning** | Aₖ_reason | Writer（入力事実から推論した新たな洞察） | 入力に明示されない因果関係・提案の出現 |

```
Aₖ = Aₖ_prior + Aₖ_tool + Aₖ_reason
```

### 3.3 Aₖ の操作的測定

**Aₖ_tool（最も測定しやすい）**:
Researcherエージェントの場合、Web検索のAPI応答が記録されている。
検索結果に含まれる情報（entity、数値）のうち、Researcherの出力に現れたものがAₖ_tool。

**Aₖ_prior（測定困難）**:
CEOの出力に含まれるentity/数値のうち、入力にもtool結果にも含まれないもの。
しかし、LLMが「学習データから知識を引き出した」のか「ハルシネーションした」のかを区別するのは困難。

→ **実用的対策**: Aₖ_priorは「出力のうち入力由来でない情報」として定義し、品質への正負の寄与は別途評価する（正: 正確な知識の付加、負: ハルシネーション）。

**Aₖ_reason（最も測定困難）**:
入力に明示されない推論結果（「したがって○○戦略が最適」等）。
自然言語の推論を自動検出するのは現在の技術では困難。

→ **実用的対策**: Aₖ_reasonは当面は測定対象から除外し、Aₖ ≈ Aₖ_prior + Aₖ_tool として操作化する。

### 3.4 C1との接続

CEOのCR=1.85（入力より多く出力する）は大きなAₖを示唆。
しかし、CEOのAₖの多くはAₖ_prior（学習済み知識の付加）であり、これが品質に正に寄与しているかは品質スコアとの相関で間接的にしか測れない。

CEO input tokens vs OQ の正相関（r=+0.207）は、「CEOへの入力が多い → CEOがそれを種にしてより良いAₖ_priorを生成する」と解釈できる。つまり**Aₖ_priorは入力量の増加関数**:

```
Aₖ_prior ≈ g(|Xₖ|)    where g は単調増加（飽和まで）
```

## 4. D_k, A_k と3つの動作モードの対応

| モード | Dₖ | Aₖ | Net = Aₖ - Dₖ | 例 |
|--------|-----|-----|---------------|-----|
| **Preservation** | ≈ 0 | ≈ 0 | ≈ 0 | coordinator forwarding |
| **Compression** | > 0 | ≈ 0 | < 0 | TL (CR=0.35): D≈0.65, A≈0 |
| **Augmentation** | small | large | > 0 | CEO (CR=1.85): D≈small, A≈large |
| **Saturation** | ≫ 0 | ≈ 0 | ≪ 0 | CFO/CDO mesh (CR=0.06): D≈0.94, A≈0 |

**Saturation はCompression の極端なケース**だが、質的に異なる:
- Compression: エージェントが選択的に情報を保持（intelligent filtering）
- Saturation: エージェントが入力を処理しきれず、ほぼランダムに情報を落とす

判定基準:
- Compression: Dₖ_filter >> Dₖ_noise（意図的フィルタリングが支配的）
- Saturation: Dₖ_noise >> Dₖ_filter（処理限界による非選択的損失が支配的）

## 5. 情報保存率 P(k) の定式化

ステップkまでの情報保存率を以下で定義:

```
P(k) = |Preserved(k)| / |T(X)|
```

Modified DPIを再帰的に展開すると:

```
P(k) = P(0) - Σᵢ₌₁ᵏ Dᵢ + Σᵢ₌₁ᵏ Aᵢ
     = 1 - Σ Dᵢ + Σ Aᵢ    (P(0)=1 と正規化)
```

**エージェントチェーン全体の品質は、P(final)と正の相関を持つと予測**:

```
OQ ∝ P(final) = 1 - Σ Dᵢ + Σ Aᵢ
```

### 5.1 各構成でのP(final)の予測

**deep構成** (CEO→TL→Researcher→Writer→Review→CEO):
```
P(final) ≈ 1 - D_CEO1 - D_TL - D_Res - D_Wri - D_Rev + A_CEO1 + A_Res + A_CEO2
```
- D_TL ≈ 0.65（TLの定率フィルタ）が主要な損失源
- A_Res は外部知識で部分的に補填

**flat構成** (CEO→Researcher/Writer(直接)→Review→CEO):
```
P(final) ≈ 1 - D_CEO1 - D_Res - D_Wri - D_Rev + A_CEO1 + A_Res + A_CEO2
```
- **D_TL項が消える** → P(final) が flat > deep を予測
- C1の結果（depth最強予測変数）と整合

**mesh構成**:
```
P(final) ≈ 1 - D項 + A項 + R項
```
- R項（Redundancy bonus）: 冗長経路による情報回復
- ただしreviewerが飽和してもR項に影響しない（主経路外）

## 7. 残された理論的課題

### 7.1 CR ≠ 情報量の問題

CR（トークン比）はDₖの代理指標に過ぎない。TLがCR=0.35でも、**重要な35%を選択的に保持**していれば、Dₖ_filter大・Dₖ_noise小であり、品質への影響は軽微。

→ **対策**: CRを「量的フィルタリング」の指標、TSR/EPRを「質的フィルタリング」の指標として、両方を報告する。理論ではDₖ = Dₖ_filter + Dₖ_noise と分解し、実験ではTSRでDₖ全体を、EPR/SSDで Dₖ_noise の寄与を推定する。

### 7.2 Aₖ_prior の測定困難性

CEOの「学習済み知識の付加」は測定困難。出力に入力由来でない情報があっても、それが「正確な知識」か「ハルシネーション」かを自動判定できない。

→ **対策**: 本論文ではAₖを「source-agnostic」に定義する（源泉を区別しない）。Aₖの品質（正確性）は品質スコアとの相関で間接的に評価。ハルシネーション vs 正確な知識の分離はFuture Work。

### 7.3 TIU間の独立性仮定

TIUを独立な単位として扱っているが、実際には情報間に依存関係がある（「市場規模3,847億円」と「年成長率25%」は合わせて意味を持つ）。

→ **対策**: 本論文では独立性を仮定し、TIU単位の生存率で分析する。情報間依存の扱いはFuture Work。

### 7.4 定率減衰モデルの妥当性

P(k) ≈ αⁿ という定率減衰は、各ステップの Dₖ/Aₖ が一定であることを仮定する。実際にはエージェントの役割（TL vs Writer vs Researcher）によって大きく異なる。

→ **対策**: 定率減衰は「同一役割エージェントのチェーン」に対する近似。異なる役割のチェーンでは、各ステップのDₖ/Aₖを個別に推定する。実験Bでは可能な限り同一役割のchainを構成して純粋な定率減衰を検証する。
