## 主要な発見

### 発見1: Critical Path Depth が最強の品質予測変数

**r = -0.255, p < 0.0001** — 主経路上のエージェント数が品質を最もよく予測する。

- flat構成 (depth=3): 平均OQ = 3.26
- deep構成 (depth=4): 平均OQ = 2.80

これはDPIの予測と整合する: **中継段数が多いほど情報が失われる**。ただし、情報量（CR）よりもdepthという**構造的変数**の方が予測力が高い点は重要。

### 発見2: CEO Input Tokens が2番目に強い予測変数

**r = +0.207, p = 0.0009** — CEOの最終統合ステップに届く情報量が多いほど品質が高い。

ボトルネック仮説（CEOへの情報過多が品質低下を招く）は**棄却**される。現在のモデル（Haiku, 200kコンテキスト）ではCEOのチャネル容量は十分に大きく、入力増加はむしろ品質を改善する。

**理論的解釈**: CEOは**情報増幅チャネル**（CR=1.85）として機能し、より多くの原材料（=入力情報）からより良い最終出力を生成する。ボトルネックはCEOではなく、CEOに情報を届ける**上流のフィルタリングノード**にある。

### 発見3: TL（中間管理層）の情報フィルタリングが品質低下の主因

**TL CR vs OQ: r = +0.169, p = 0.016** — TLの情報保持率が高いほど品質向上。

TLは平均CR=0.35で入力の65%をカットする。この過度なフィルタリングが、CEOに届く情報量を減らし、最終品質を低下させている。

**因果メカニズムの仮説**:
```
deep構成: Task → CEO → TL(CR=0.35) → Researcher → Writer → Review → CEO(統合)
  → TLが情報を65%カット → CEOに届く情報が少ない → 品質低下

flat構成: Task → CEO → Researcher/Writer(直接) → Review → CEO(統合)
  → TLなし → CEO入力が多い → 品質向上
```

### 発見4: Bottleneck Principle の修正が必要

当初のBottleneck Principle（「最小CRのエージェントがボトルネック」）は**そのままでは成立しない**。

理由: **全てのエージェントが同じ「主経路」上にいるわけではない**。

- CFO/CDO（レビュアー）: 主経路外。彼らのCR低下（飽和）は最終品質に直接影響しない
- flat_meshのCFO/CDOはCR=0.06だが、品質は最高（3.30）

**修正版 Bottleneck Principle**:

> 組織全体の情報スループットは、**情報の主経路**（タスク入力→最終出力に至る経路）上の最小容量ノードによって制約される。主経路外のエージェント（レビュアー等）の飽和は、主経路の情報フローに直接影響しない。

さらに、min(CR)よりも**depth（中継段数）**の方が予測力が高い。これは各段の情報損失が**ほぼ一定率**（TL≈0.35で固定）であることを示唆し、**定率減衰モデル**を支持する:

```
I(X; Y_n) ≈ I(X; Y_0) × α^n   where α ≈ 0.35 (per TL relay)
```

## 論文への反映

### Sec 3.5 (Bottleneck Principle) の修正

Proposition 3 を以下に修正:

**Revised Bottleneck Principle**: 情報スループットは主経路上の最小容量ノードで制約される。ただし、depth（主経路長）自体が最も強い予測変数であり、min(CR)よりも品質をよく予測する。これは各中継段での情報損失率がほぼ一定（定率減衰）であることを示唆する。
